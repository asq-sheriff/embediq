# Getting started with EmbedIQ

A ten-minute guided tour: install EmbedIQ, run the wizard against a
throwaway project directory, look at what it generated, and drift-check
the result. If you're comfortable at the end, jump to the
[user-guide index](user-guide/) for feature-specific walkthroughs.

## Prerequisites

- Node.js 18 or newer (`node --version`)
- A scratch directory for the generated files — a new empty git repo
  works, or just any folder you can delete later.

## 1. Install

```bash
git clone https://github.com/asq-sheriff/embediq.git
cd embediq
npm install
```

Optional: `make check` runs the full type-check + test suite to confirm
your local environment is clean before you start.

## 2. Run the CLI wizard

```bash
npm start
```

The wizard walks through four phases:

1. **Discovery** — ~25–40 adaptive questions across seven dimensions.
2. **Playback** — summary of the profile + derived priorities.
3. **Edit & approve** — correct any field before generation.
4. **Generate** — writes the files to a directory you choose.

For a quick first run, answer these minimum questions and accept
defaults for the rest:

| Question                 | Answer                |
| ------------------------ | --------------------- |
| Your role                | `developer`           |
| Technical proficiency    | `intermediate`        |
| What are you building    | `Web application`     |
| Industry                 | `saas`                |
| Team size                | `solo`                |
| Primary language         | `typescript`          |
| Budget tier              | `moderate`            |
| Regulated sensitive data | `no`                  |

When prompted for a target directory, pick a scratch folder — e.g.
`/tmp/embediq-demo`.

## 3. Inspect the output

```bash
cd /tmp/embediq-demo
ls -la
```

You should see:

```
CLAUDE.md
.claude/
  settings.json
  settings.local.json
  rules/
    testing.md
    typescript.md
  hooks/
    command-guard.py
.claudeignore
.mcp.json.template
```

Open `CLAUDE.md`. It's a summary of your tech stack, conventions, and
workflow — Claude Code reads this as its system prompt every session.
See [`user-guide/02-generated-files.md`](user-guide/02-generated-files.md)
for a tour of every file the synthesizer can emit.

## 4. Drift-check the output

The wizard is deterministic. Any later edit to a generated file should
show up as drift:

```bash
# From the embediq repo root
echo "# modified by hand" >> /tmp/embediq-demo/CLAUDE.md

npm run drift -- \
  --target /tmp/embediq-demo \
  --archetype minimal-developer
```

You'll see:

```
EmbedIQ Drift Report
  Status:  DRIFT
  Target:  /tmp/embediq-demo

  Modified by user (no EmbedIQ stamp) (1)
    ! CLAUDE.md
        File exists at target but was not generated by EmbedIQ (no stamp)
```

Exit code is `1` — suitable for gating a CI job on a clean state. Full
drift documentation: [`user-guide/06-evaluation-and-drift.md`](user-guide/06-evaluation-and-drift.md).

## 5. Try multi-agent targets

The same interview can produce configuration for Cursor, Copilot,
Gemini, and Windsurf in addition to Claude Code. Re-run with a
target list:

```bash
EMBEDIQ_OUTPUT_TARGETS=claude,cursor,agents-md npm start
```

After generation, you'll see `AGENTS.md` at the repo root and a
`.cursor/rules/` tree alongside the `.claude/` tree. See
[`user-guide/05-multi-agent-targets.md`](user-guide/05-multi-agent-targets.md)
for the full target matrix and per-target details.
